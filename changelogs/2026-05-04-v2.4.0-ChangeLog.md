# 2.4.0 – 2026-05-04

Bidirectional sync feature branch — standardized Scryfall data extraction and MTG Cube Maker integration.

### Added

#### Scryfall — Enhanced Data Extraction
- Extracts `frame_effects` from Scryfall API (showcase, extended art, legendary, etc.)
- Captures `full_art` boolean flag for full-art cards
- Extracts `promo_types` for promotional card variants
- Stores complete `scryfall_data` JSON blob for future extensibility
- Extracts `legalities` object for format legality (commander, modern, legacy, etc.)
- Properly handles "etched" finish type alongside foil/nonfoil

#### MTG Cube Maker — Bidirectional Sync
- Cards synced from MTG Cube Maker now include all rich MTG data fields
- Proxy cards are treated as distinct variants (is_proxy flag preserved)
- Duplicate detection considers proxy status — proxy and non-proxy cards don't merge

### Changed

#### Scryfall — Variant Standardization
- Variant field now consistently set to finish type: "foil", "nonfoil", or "etched"
- Previously variant was blank in some cases causing duplicate detection issues
- Now matches MTG Cube Maker's variant handling for consistent sync

#### Local Fallback Cache
- Updated `find_mtg_cards_local()` to extract `frame_effects` and `promo_types` from stored JSON
- Added `has_etched` field detection
- Variant properly derived from prices (etched → foil → nonfoil priority)

### Backend

- `external/scryfall.py` — Added `frame_effects`, `full_art`, `promo_types`, `scryfall_data`, `legalities` to search results
- `models/card.py` — Added new columns: `frame_effects`, `full_art`, `promo_types`, `scryfall_data`, `legalities`
- `schemas/card.py` — Added new fields to `CollectionCardBase` and `MtgCardSummary`
- `services/collection_service.py` — Saves all new fields on card create/update; includes `is_proxy` in duplicate detection
- `legacy/fallback_manager.py` — Updated `store_mtg_card()` and `find_mtg_cards_local()` with new fields
