# 2.2.1 – 2026-04-06

Import / Export improvments

### Added

#### Import — Real-time Progress Bar
- Import now streams progress card-by-card via Server-Sent Events (SSE) instead of waiting for a single blocking response.
- A progress bar appears as soon as the upload completes, filling smoothly as each row is processed.
- The label below the bar shows the current card name and a contextual status icon:
  - 🔍 **Fetching** — looking up a Scryfall ID directly
  - 🔍 **Searching** — running fuzzy name/set search
  - ✓ **Imported** — card added successfully
  - ⚠ **Needs review** — multiple candidates, will appear in the disambiguation queue
  - — **Skipped** — row had no name
  - ✕ **Error** — row failed to insert
- A `current / total (%)` counter is shown on the right side of the bar.
- Bar color shifts to amber on ambiguous cards and red on errors.
- New `POST /api/import/csv/stream` endpoint backs the feature; the original `/api/import/csv` endpoint remains for non-streaming callers.

#### Import History — Previous Imports Tab
- New **📋 Previous Imports** inner tab on the Import/Export screen (alongside the existing Import / Export tab).
- Lists every import batch with: filename, game, cards added, pending disambiguation count, and timestamp.
- **⬇ Download** button re-serves the exact original CSV that was uploaded, stored as binary in the database.
- **✕ Undo** button deletes the import batch:
  - Removes all `CollectionCard` rows created by that batch (using stored card IDs).
  - Clears any still-pending ambiguities linked to the batch.
  - Removes the history record itself.
  - Invalidates the collection cache so the card count updates immediately.
- A confirmation dialog shows the exact card count before any deletion.
- History auto-loads when the tab is opened; a ↺ Refresh button allows manual reload.
- Resolved ambiguities (cards added after the initial import via the disambiguation UI) are correctly tracked in the batch and removed on undo.

### Changed
- `import_csv_stream()` generator is now the canonical implementation; `import_csv()` is a thin wrapper that drains the generator.
- `resolve_ambiguities()` now appends resolved card IDs to `ImportHistory.created_card_ids` and keeps `imported_count` / `ambiguous_count` accurate as the user works through the disambiguation queue.
- `/api/import/csv` endpoint now passes `file.filename` through to the service so the original filename is stored in history.

### New Endpoints
| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/import/csv/stream` | SSE progress stream for CSV import |
| `GET`  | `/api/import/history/{owner_id}` | List import history (no file bytes) |
| `GET`  | `/api/import/history/{owner_id}/{id}/file` | Download original import CSV |
| `DELETE` | `/api/import/history/{owner_id}/{id}` | Undo import, delete all created cards |

## [2.2.0] – 2026-03-28

### Added

#### Import / Export Scoping
- Import/Export tab is now hidden when the **All** scope is selected — it only appears when a specific collection type (MTG, Pokémon, Sports Cards, Collectibles) is active.
- Switching back to "All" automatically redirects away from the Import/Export tab.

#### Manabox Import Support
- MTG imports now handle Manabox CSV exports natively.
- `foil`/`normal` and `altered`/`normal` string values from Manabox are mapped automatically without requiring the header mapper.
- Scryfall ID column enables fast, unambiguous matching — no disambiguation queue for cleanly exported Manabox files.

#### Manabox Export
- New **⬇ Export for Manabox** button (MTG only) generates a Manabox-compatible import CSV.
- Backed by `POST /api/export/manabox`.

#### Flexible Header Mapper
- On file selection, the importer reads the CSV headers and presents a two-column mapping grid: **Target Field** → **Source Column**.
- ~40 built-in aliases auto-map common column name variants (e.g. `qty` → Quantity, `player` → Name, `cert` → Serial Number).
- An **Auto-detect** button re-runs the auto-mapping at any time.
- Mapped fields are highlighted in green; unmapped fields remain neutral.
- Applies to all collection types (MTG, Pokémon, Sports Cards, Collectibles).

#### Import Templates & Field Guides
- Collapsible `<details>` field guide blocks added for every collection type:
  - **MTG** — standard format guide + separate Manabox format reference
  - **Pokémon** — field descriptions, variant notes
  - **Sports Cards** — grading fields, insert/parallel, rookie card flag
  - **Collectibles** — series, manufacturer, UPC, condition
- Each guide includes a **⬇ Download blank template** button that generates a ready-to-fill CSV.

#### Sports Cards & Collectibles Import
- Importer now fully supports Sports Cards and Collectibles via the header mapper.
- These types skip the search API path and insert directly, which also removes the disambiguation queue for them.
- New fields populated on import: `sport`, `grading_company`, `grade`, `serial_number`, `print_run`, `rookie_card`, `manufacturer`, `upc`.

#### Build & Refresh Sidebar Button
- Lower-left sidebar now contains a **Build & Refresh** button.
- Triggers a 5-phase sequence: build frontend → restart backend → wait for server → reload page.
- Phase indicator shows live status; errors surface two recovery options: ↺ Refresh page or ✕ Dismiss.
- Restart logic is uvicorn-reload-aware: sends `SIGTERM` to the worker (letting the supervisor respawn it) instead of `os.execv` which would conflict on the bound port.

### Backend
- `_apply_mapping()` in import service renames source CSV columns to target display names before normalization.
- `_row_to_card_data()` extended to populate all Sports Cards and Collectibles fields.
- Search gating fixed: Sports Cards and Collectibles correctly skip the MTG/Pokémon search APIs.
- `ImportHistory` model added with `LargeBinary` file storage, JSON card ID list, and owner FK cascade.
- `ImportAmbiguity` model extended with `import_history_id` FK.
- `POST /api/dev/rebuild` and `POST /api/dev/restart` endpoints added (`dev` router).
