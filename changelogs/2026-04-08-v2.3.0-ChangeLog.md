# 2.3.0 – 2026-04-08

Coins feature branch — full numismatic support with NGC integration, per-coin imagery, and strike-type grading.

### Added

#### Coins — NGC S3 Images
- Per-coin obverse and reverse images sourced directly from the NGC S3 CDN (`ngccoin-production/us-coin-explorer/`).
- Discovered the NGC internal API (`/coin-explorer/data/coins/{coinID}/`) that returns `ObverseImageURL` and `ReverseImageURL` without JavaScript rendering.
- Replaced prior Wikimedia Commons approach which was returning incorrect or generic images (e.g. 1879-S Morgan for 1921 results).
- Images are cached in-process on first fetch and persisted to the local CSV cache so subsequent searches return them instantly.

#### Coins — Coin Flip (Obverse / Reverse)
- Search result cards now show a 🪙 flip button when a reverse image is available, mirroring the DFC flip feature for MTG cards.
- Clicking toggles between the obverse and reverse of the coin.

#### Coins — Strike Type Selector (MS / MS PL / MS DPL)
- NGC lists up to three strike variants per year/mint: **MS** (Mint State), **MS PL** (Proof-Like), and **MS DPL** (Deep Mirror Proof-Like). Proof coins follow the same pattern with **PF**, **PF Cameo**, and **PF Deep Cameo**.
- Search results now show **one card per year/mint** instead of three separate rows, with pill-style buttons to switch between types.
- Selecting a type updates the grade pills, price display, NGC link, and coin images to reflect that specific variant.
- Defaults to **MS** on every new result.

#### Coins — Grade & Strike Type Saved Separately
- Adding a coin to the collection stores the strike type as `variant` (e.g. `MS PL`) and the selected NGC grade as `grade` (e.g. `MS-63`), keeping the two concepts independent.

#### Coins — Accurate Collection Links
- **Name** column in My Collection links to the specific coin's NGC Coin Explorer page.
- **Series** column links to the NGC Price Guide series page.

#### Coins — Search Result Info Block
- Updated the info block below each coin image: `Morgan Dollar 1886 / Mint: O / NGC Link ↗`
- Removed the raw series name/URL clutter from the previous layout.

#### Coins — Mint Mark Fix
- Mint mark was not being saved to the database when adding a coin to the collection. Fixed mapping in `collection_service.add_card()`.

#### Settings — About Tab
- New **About** tab in Settings showing the current app version, a brief feature description, tech-stack build info, and a link to [y2cl.net](https://y2cl.net).

### Changed

#### Coins — Edit Modal Relabeled for Numismatics
- Field labels updated so coins no longer show sports-card-centric names:
  - **Name** → **Coin Name**
  - **Card #** → **Date / Variety**
  - **Set** → **Series**
  - **Insert** → **Strike Type**
- **NGC Grade** field added to the Coin Details section (displays the stored grade, e.g. `MS-63`).
- `grade` field is now saved and loaded for coins in the edit modal (previously only saved for Sports Cards).

#### Coins — My Collection Grade Column
- The **Grade** column for coins now displays the `grade` field (e.g. `MS-63`) instead of the `variant`/strike type field.
- Falls back to `variant` for older records saved before the grade field was introduced.

### Backend

- `ngc_coins._fetch_ngc_coin_images()` — new function that calls the NGC coin explorer JSON API and caches results in `_NGC_COIN_IMG_CACHE`.
- `ngc_coins._coin_type_label()` — maps strike type + row position to a human label (MS / MS PL / MS DPL etc.).
- `ngc_coins._parse_price_table()` — tracks position within each `(date_label, strike_type)` group to assign `coin_type` to every coin row.
- `ngc_coins.search_ngc_coins()` — merges rows with the same year/mint into a single result with `coin_type_options` and `coin_types_data` fields.
- `schemas/card.py` — `CardResult` extended with `coin_type_options` and `coin_types_data`.
- `fallback_manager.py` — `coin_type_options` and `coin_types_data` persisted to CSV as JSON strings.
- `SearchResultsGrid.tsx` — cards now keyed by `card.link` to guarantee remount (and correct default state) when results change.
