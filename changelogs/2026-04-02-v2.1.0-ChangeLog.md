# v2.1.0 — 2026-04-02

PR #5 / #6 — Single run command, collection page improvements, import overhaul, owner/profile management.

## Added

### Infrastructure
- Single `npm run dev` command at the project root runs backend and frontend concurrently via `concurrently`
- `README.md` with full setup and run instructions
- `.nvmrc` for Node version pinning

### Collection Page
- Bulk move cards between owners/profiles
- Bulk price refresh — re-fetches current prices from Scryfall (MTG) or Pokémon TCG API
- Owner and profile rename and delete in Management tab
- Profile update and delete endpoints added to API

### Import
- Overhauled CSV import service: full column normalization, variant derivation, signed/altered handling
- Scryfall enrichment on import for MTG cards (backfills images and prices from `scryfall_id`)
- Import deduplication respects the configured duplicate strategy (merge/separate)
- Ambiguity queue: cards that can't be auto-resolved are held for manual review
- `DELETE /api/collection/import/ambiguities/{owner_id}` endpoint to clear the queue

### API
- `POST /api/collection/cards/move` — bulk move cards to another owner/profile
- `POST /api/collection/cards/refresh` — bulk price refresh for selected cards
- `PUT/DELETE /api/owners/{id}` — rename and delete owners
- `PUT/DELETE /api/owners/{id}/profiles/{id}` — rename and delete profiles
- Export endpoint for collection data

## Fixed
- Scryfall double-faced card image sync on prop change (useEffect re-sync)
